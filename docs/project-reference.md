# Project Reference

Use this page as the quick implementation guide when referencing McGuire Technology™ from other projects.

## How to Reference the Brand

First prominent use:

> McGuire Technology™

Repeated body-copy use:

> McGuire Technology

Company/legal entity use when needed:

> McGuire Technology, LLC

## README Pattern

Recommended footer or attribution block:

> Built by McGuire Technology™.
>
> McGuire Technology™ builds practical software platforms and technical systems for operational clarity, automation, governance, and reliability.

## Project Language

For projects that are part of the broader brand:

> This repository is part of the McGuire Technology™ ecosystem and follows the McGuire Technology brand, documentation, and trademark guidance.

## Product Page Pattern

Use this structure for product or project pages:

1. Product or project name
2. One-line operational value
3. The problem it clarifies or controls
4. Key capabilities
5. Where it fits in the McGuire Technology ecosystem
6. Trademark or licensing note when needed

## Naming Checklist

A product name should:

- be easy to say in a business conversation
- describe or suggest a durable capability
- work as part of a product family
- avoid trend-dependent terminology
- avoid names that sound too consumer, playful, or gimmicky

Good naming territories:

- Assets
- Controls
- Observe
- Orchestrate
- Ledger
- Identity
- Flow
- Nexus

## Visual Checklist

When creating a project page or UI, use:

- shadcn/ui as the preferred UX framework for React and Next.js interfaces
- Ubuntu Sans for primary text
- a restrained neutral base
- Signal Blue for primary actions or active states
- Teal for success or trust signals
- Amber for warnings or attention
- clear tables, panels, and diagrams
- screenshots or diagrams when they explain the product
- semantic tokens and component variants before one-off custom styling

Avoid:

- heavy gradients
- generic SaaS art
- decorative stock photos
- copy that promises more than the product can show
- unrelated color palettes for every project
- custom controls that duplicate standard shadcn/ui components
